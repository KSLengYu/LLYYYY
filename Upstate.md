## 更新日志 1.0
- 修复：回复现在正确作为父留言的子回复保存，并在首页支持折叠/展开显示回复。
- 新增：个人主页可以重新绑定 QQ（解绑后可再次绑定）。
- 新增：游客模式入口恢复（每天默认限制 5 条，可配置）。
- 改进：非登录用户也能看到用户发布留言时的头像与昵称（由后端返回）。
- 改进：密码设置/修改改为站内弹窗（替代浏览器 prompt），体验更连贯。
- 改进：登录/注册页把发送验证码与输入验证码合并为单页流程；验证成功后有渐变线性进度条提示。
- 新增：留言附带网络类型（WiFi/2G/3G/4G/5G 等）与设备机型信息（best-effort），方便管理端查看来源。
- 界面：首页、后台与个人页进行了视觉优化，风格更“科技/高大上”。
- 安全：后台新增主管理员（superadmin）角色，用于授权其它管理员（请通过 SQL 将你的邮箱设置为 superadmin）。

## 更新日志 1.0.1
- 修复：回复将作为目标留言的 reply（保存 parent_id），不会再变成新贴。
- 功能：个人主页绑定/解绑 QQ，解绑后回显“绑定 QQ”按钮以便重新绑定。
- 功能：游客模式（每天默认允许 5 条，环境变量 GUEST_DAILY_LIMIT 可配置）。
- UI：登录/注册合并 OTP 步骤，内置科幻风格 modal + 渐变进度条，替换浏览器原生 prompt（部分简单 prompt 仍保留）。
- 设备/网络：发帖时会尝试检测并保存设备型号与网络（WiFi / 5G/4G/3G），并显示在留言下方。
- 管理：管理后台可通过邮箱或 QQ 查找用户并封禁、解封、设置/取消管理员（需管理员权限）。
- 后端：合并 utils 到 api/v1.js，所有 API 路由统一为 `/api/v1/<route>`。
- 安全：密码限制仅允许英文与数字，长度至少 6 位（前端/后端均校验）。
